利用Git和Hexo搭建博客

date: 2017-03-13 10:29:59
tags:技术

安装及新建 hexo

官方推荐安装方法：
curl -sL https://deb.nodesource.com/setup_6.x | sudo -E bash -
sudo apt-get install -y nodejs //因为hexo是基于nodejs的应用，所以需要先安装nodejs
sudo npm install hexo-cli -g //安装hexo
hexo init “listname” //新建博客目录
cd listname //进入博客工作目录
npm install //安装npm
hexo server //开启本地hexo服务，预览当前博客

注：
_config.yml是配置文件，
例：
title: 目之所及 //标题
subtitle: 技术学习 //副标题
description:
author: kaven //作者
language: zh-CN
public是hexo生成的静态站点文件夹，
scaffolds是模板文件夹，
source是存在用户资源的文件夹，
themes是主题文件夹。

hexo new "文章标题"    //新建文章，生成位置：sources/_posts 编辑该文件创作博客

注册安装 github

makdir ~/hello-world    //创建一个项目hello-world
cd ~/hello-world    //打开这个项目
git init        //初始化，建立本地仓库 
git commit -m 'first commit'     //提交更新，并注释信息“first commit”
git remote add origin git@github.com:defnngj/hello-world.git     //连接远程github仓库项目
git push -u origin master    //将本地项目更新到github项目上去 -u为首次更新 或者（+master）为添加更新

注：
Git有一个工具被称为git config，它允许你获得和设置配置变量；这些变量可以控制Git的外观和操作的各个方面。这些变量可以被存储在三个不同的位置：
1 /etc/gitconfig 文件：包含了适用于系统所有用户和所有库的值。如果你传递参数选项’–system’ 给 git config，它将明确的读和写这个文件。
2 ~/.gitconfig 文件 ：具体到你的用户。你可以通过传递–global 选项使Git 读或写这个特定的文件。
3 位于git目录的config文件 (也就是 .git/config) ：无论你当前在用的库是什么，特定指向该单一的库。每个级别重写前一个级别的值。因此，在.git/config中的值覆盖了在/etc/gitconfig中的同一个值。
配置相关信息：
1 当你安装Git后首先要做的事情是设置你的用户名称和e-mail地址。这是非常重要的，因为每次Git提交都会使用该信息。它被永远的嵌入到了你的提交中：
git config –global user.name “John Doe”
git config –global user.email johndoe@example.com
2 检查你的设置(Checking Your Settings)
　　 如果你想检查你的设置，你可以使用 git config –list 命令来列出Git可以在该处找到的所有的设置:
git config –list
你也可以查看Git认为的一个特定的关键字目前的值，使用如下命令 git config {key}:
git config user.name
3 获取帮助(Getting help)
如果当你在使用Git时需要帮助，有三种方法可以获得任何git命令的手册页(manpage)帮助信息:
git help
git –help
man git-
例：
git help config //获取帮助

设置ssh连接

git config --global user.name "yourname" //设置git用户名
git config --global user.email“your@email.com" //设置git邮箱
ls -al ~/.ssh        //查看.ssh目录
rm known_hosts
            // 删除.ssh文件夹（直接搜索该文件夹）下的known_hosts
ssh-keygen -t rsa -C "your@email.com"  //填写设置的邮箱地址

系统会自动在.ssh目录下生成两个文件，id_rsa和id_rsa.pub，用编辑器打开id_rsa.pub
将全部的内容复制
打开https://github.com/，登陆你的账户，设置，ssh设置
将复制的key粘贴进去
点击add ssh key
ssh -T git@github.com /测试连接是否成功
若未成功
eval “$(ssh-agent -s)”
ssh-add

部署 hexo

npm install hexo-deployer-git –save //装插件
修改_config.yml中的deploy参数，添加
branch： master；
repository: git@github.com:nelifew/nelifew.github.io.git
hexo generate -deploy(可以简化为hexo g -d) 生成推送到github的master分支
此时访问your-user-name.github.io即可查看生成的站点内容

依次执行
git add .
git commit -m “…”
git push origin
hexo g -d //发布
